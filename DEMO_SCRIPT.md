# NUS Campus Intelligent Assistant — Demo Script

---

## Opening (30 seconds)

"Hi everyone. What I'm going to show you today is the **NUS Campus Intelligent Assistant** — a multi-agent AI system we built specifically for NUS students.

The core idea is simple: a student's campus life involves a lot of repetitive, time-consuming tasks — tracking assignments, managing schedules, applying for jobs, navigating campus. We built an AI agent system that handles all of that automatically.

The system is powered by three technologies: **OpenClaw** for multi-agent orchestration, **WaveSpeed AI** for LLM and Vision inference, and **ChromaDB** for our RAG knowledge base.

Let me walk you through each feature."

---

## Feature 1 — Knowledge Q&A (1 minute)

*[Click on Knowledge Q&A in the sidebar]*

"This is our RAG-based campus knowledge assistant. Under the hood, we've built a **hybrid retrieval pipeline** — combining ChromaDB semantic vector search with BM25 keyword search, fused together using Reciprocal Rank Fusion. The answer is then generated by WaveSpeed AI using Gemini 2.5 Flash.

Let me ask it something."

*[Type: "What are the PhD requirements for Computer Science at NUS?"]*

"You can see it's streaming back a detailed answer, and at the bottom it cites the actual source URLs it retrieved from.

But here's what makes it more than just a chatbot — it has **agent intent detection**. If I ask it something like 'sync my Canvas' or 'help me find a job', it doesn't just answer — it recognizes the intent and routes me to the right agent automatically."

---

## Feature 2 — Schedule Agent (1.5 minutes)

*[Click on Schedule Agent in the sidebar]*

"This is our Schedule Agent. It solves a real pain point — NUS students have assignments scattered across Canvas, and exam dates buried in PDF syllabi. We unify everything into one smart calendar.

**First — Canvas Sync.**"

*[Enter Canvas token and click Sync]*

"With one click, the agent calls the Canvas LMS API, pulls all assignments and announcements, and WaveSpeed AI automatically summarizes each announcement so you don't have to read the full HTML."

*[Point to the timeline]*

"You can see all events here color-coded by urgency — red means due within 3 days.

**Now — Syllabus Vision.**"

*[Drag a syllabus PDF into the upload area]*

"I'm uploading a course syllabus PDF. Watch what happens — the agent converts each page to an image, sends it to WaveSpeed Vision AI, and extracts every exam date, deadline, and project milestone automatically."

*[Show the extracted events appearing]*

"It extracted all of those in seconds, from a raw PDF. No manual entry.

**Finally — the calendar sync.** Instead of downloading a file and manually importing it, we have a true agent action: one click connects to Google Calendar via OAuth, and the agent automatically creates every single event with 24-hour email reminders and 1-hour popup alerts already embedded."

---

## Feature 3 — Job Agent (1.5 minutes)

*[Click on Job Agent in the sidebar]*

"This is our Job Agent. The problem we're solving: applying for internships is incredibly tedious. Students spend hours searching, reading job descriptions, writing cover letters, and filling out forms.

Our agent automates the entire pipeline."

*[Show the resume already uploaded, keywords filled in]*

"I've uploaded my resume as a PDF — WaveSpeed AI already parsed it into structured JSON: name, skills, education, work experience, projects.

Now I hit **Launch Agent** and watch the Agent Console."

*[Click Launch Agent]*

"The agent is now:
- Searching LinkedIn for matching positions
- For each job, WaveSpeed AI reads the job description and compares it against my resume, giving a match score from 0 to 100
- Any job scoring above my threshold gets auto-applied via Playwright browser automation

You can see the real-time log: each job analyzed, each score, each application submitted. This whole process — which normally takes hours — runs automatically."

*[Point to the results]*

"Three applications submitted, scored 78, 74, and 71 respectively, all saved to the application history."

---

## Feature 4 — Campus Route Planner (1.5 minutes)

*[Click on Campus Route in the sidebar]*

"Our fourth agent is the Campus Route Planner. This one is particularly interesting architecturally because it uses a **three-skill agent pattern**.

Let me give it a real scenario."

*[Click the quick prompt: "我今天要办理宿舍入住，请帮我规划路线"]*

"I'm asking it to plan my dormitory check-in route. Watch the agent reasoning in real time:"

*[Point to the processing steps as they appear]*

"**Skill 1 — get_calendar_events**: The agent reads my calendar and finds I have a class at 2pm today. It flags that as a potential conflict.

**Skill 2 — search_knowledge_base**: It queries our RAG system for dormitory check-in procedures — which offices to visit, what documents to bring, how long each step takes.

**Skill 3 — calculate_route_matrix**: It computes the optimal walking and bus times between each stop — Blk 365 to UTown to EA to YIH to Kent Ridge Hall.

Then WaveSpeed AI synthesizes all of this into a natural language summary and builds a full timeline."

*[Point to the map]*

"On the left, you can see the route drawn on the map in real time. Each pin is color-coded — blue for the starting point, yellow for service offices, green for the destination.

On the right, each timeline card shows the exact time, transport mode, what to do at each stop, office hours, and importantly — this purple card here shows the class conflict the agent detected and automatically scheduled around."

## Closing (30 seconds)

"So to summarize — what we've built is not just a chatbot. It's a **multi-agent system** where each agent has clearly defined skills, they share a common knowledge base through RAG, and they communicate intent through a unified chat interface.

The four agents cover the four biggest time sinks in a student's campus life: **learning**, **scheduling**, **job hunting**, and **navigation**.

Everything runs locally with your own API key — no data leaves your machine except to WaveSpeed AI for inference.

Thank you."

## Quick Q&A Prep

**"What if LinkedIn blocks the bot?"**
> "We detect the block automatically and switch to demo mode — the full AI scoring and application pipeline still runs, just on curated mock data. This way the agent functionality is always demonstrable."

**"How does the RAG work?"**
> "We crawl NUS websites — graduate admissions, NUSMods, student services — chunk the content, embed it with sentence-transformers, and store it in ChromaDB. At query time we do both semantic search and BM25 keyword search, then fuse the rankings using RRF to get the most relevant documents before passing them to WaveSpeed AI."

**"Why WaveSpeed AI?"**
> "WaveSpeed AI is our hackathon sponsor and provides both LLM and Vision inference through a single unified API. We use Gemini 2.5 Flash for all text generation and vision tasks, which gives us strong multilingual support — important for NUS students who mix English and Chinese."
